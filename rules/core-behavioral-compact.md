# Core Behavioral Rules (Layer 0 — Always Loaded)
> [!NOTE]
> Trigger: Always loaded (Layer 0). Read before every task.

These 4 principles are non-negotiable. Full details in `rules/vocabulary.md` §BEH.

- **Think Before Coding `[BEH-01]`:** State assumptions, clarify ambiguities, stop & ask if confused. 
- **Simplicity First `[BEH-02]`:** Write minimal required code. No speculative abstractions/features.
- **Surgical Changes `[BEH-03]`:** Touch only requested lines. Match existing style. No drive-by refactoring.
- **Goal-Driven Execution `[BEH-04]`:** Define success criteria first as testable checks. Write test alongside code.
