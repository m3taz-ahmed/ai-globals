# Brainstorming

**Mode**: Run BEFORE writing ANY code or plan.

## 🔴 Hard Gate
Do NOT invoke implementation skills, write code, or scaffold until you have presented a design and the user approved it.

## 🧱 The Process
1. **Explore context**: Read files/docs to understand the current state.
2. **Clarifying Questions**: Ask questions ONE AT A TIME. Understand constraints/success criteria.
3. **Propose Approaches**: Give 2-3 technical options with trade-offs + your recommendation.
4. **Present Design**: Write a design spec. Get user approval.
5. **Self-Review**: Scan for "TODO/TBD" or ambiguity and fix inline.
6. **Next Step**: Once approved, move to `writing-plans`. Do NOT start implementation.
